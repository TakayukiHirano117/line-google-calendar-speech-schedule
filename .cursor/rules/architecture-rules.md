# このプロジェクトのアーキテクチャについて
オニオンアーキテクチャの思想を反映したアーキテクチャにしています。

main.tsで外部に出す関数を定義しています。
index.tsではdoGetやdoPostなどのGASでは直接関数として定義しなくてはいけない予約済み関数を定義しており、
その内部でHandlerのクラスを呼び出しています。

Handler, UseCase, Infraはオニオンアーキテクチャの概念です。

Handlerでは基本的にリクエストの受け取りとレスポンスの返却を行います。
またUseCaseはHandlerからのみ呼び出されます。
lineWebhookHandlerのみ、処理を追加する際はStrategyパターンのようにifの条件分岐でクラス呼び出しを分岐しています。

UseCase層はユースケースを定義します。
Infra層のクラスはUseCaseからのみ呼び出されます。

Infra層は外部API呼び出しなどの技術的な責務を負います。
